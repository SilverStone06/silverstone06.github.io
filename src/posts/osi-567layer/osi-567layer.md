## 0️⃣ OSI 상위 계층을 묶은 이유 ❗

OSI 5·6·7 계층은 각각 독립된 역할을 가지고 있지만, 실제 네트워크 동작에서는 서로 긴밀하게 연결되어 하나의 응용 프로그램처럼 작동합니다.

특히 현대 인터넷 구조인 TCP/IP 모델에서는 이 세 계층을 하나로 묶어 “Application Layer”로 취급하기 때문에 OSI 계층처럼 분리해서 보지 않는 경우가 많습니다.

그렇지만 OSI 관점에서 상위 계층을 한 번 명확히 정리해두면 “데이터가 전송되기 전 어떤 준비 과정을 거쳐 실제 애플리케이션에 도달하는지” 흐름을 훨씬 쉽게 이해할 수 있으니 잠깐 훑고 가시죠 !

---

## 1️⃣ OSI 7계층 — L5 : 세션 계층(Session Layer)

세션 계층은 **응용 프로세스 간 통신을 관리하기 위한 논리적 연결(세션)을 만들고 유지하고 종료하는 계층**입니다.

쉽게 말해,

> “대화창을 열고, 유지하다가, 종료하는 역할”

이라고 이해하면 됩니다.

### 세션 계층의 주요 기능

  - **세션 생성/종료**
    - TCP/IP 세션을 생성하고 통신 종료 시 세션을 닫음
  - **동기화(Synchronization)**
    - 데이터 스트림 내 체크포인트를 설정하여 통신 오류 시 특정 지점부터 재전송 가능
  - **대화 제어(Dialog Control)**
    - 통신 방향 관리(반이중/전이중 등 L1 물리 계층 방향 제어와는 다른 개념: 응용 수준의 대화 흐름 관리)
### 세션 계층에서 동작하는 대표적 기능(PDF 기반)

PDF에서는 특정 프로토콜 이름보다 **역할 중심으로 정리**되어 있었음:

  - TCP/IP 기반 세션 생성
  - 세션 유지 및 종료에 대한 논리적 정의
즉, 실제 네트워크 구동 시 **TCP 연결(TCP 3-way handshake)**, **세션 종료(4-way)** 등이 L4에서 구현되지만, 그 세션이라는 개념을 정의하는 계층이 바로 L5입니다.

---

## 2️⃣ OSI 7계층 — L6 : 표현 계층(Presentation Layer)

표현 계층은 **데이터의 형식(Format)을 정의하고 변환하는 계층**입니다. 즉, 응용 프로그램이 이해할 수 있는 형태로 데이터의 포맷을 맞추는 역할입니다.

> “데이터를 보여줄 수 있는 모양으로 변환하는 계층”

이라고 생각하면 쉽습니다.

### 표현 계층의 핵심 역할

  - **데이터 포맷 변환**
    - JPEG, GIF, PNG 등 이미지 포맷
    - ASCII, UTF-8 같은 문자 인코딩
  - **데이터 압축**
    - 전송 효율 향상
  - **데이터 암호화/복호화**
    - SSL/TLS처럼 사실상 L6 역할의 대표적인 기술
  - **전송하려는 데이터의 포맷을 결정** (GIF, JPEG, ASCII 등)
즉, 어떤 형태로 데이터를 보낼지 결정하는 곳이다.

---

## 3️⃣ OSI 7계층 — L7 : 응용 계층(Application Layer)

응용 계층은 사용자에게 가장 가까운 계층이며, **우리가 사용하는 애플리케이션이 네트워크와 맞닿는 지점**입니다.

브라우저·메일 클라이언트·SSH 클라이언트 등이 모두 L7에 해당합니다.

### 응용 계층의 주요 역할

  - 애플리케이션이 네트워크를 이용할 수 있게 하는 인터페이스 제공
  - HTTP, FTP, SSH 등 애플리케이션 프로토콜 동작
  - 사용자 요청 전달 및 결과 수신
### 주요 프로토콜 & 포트번호

| 프로토콜 | 설명 | TCP/UDP | 기본 포트 |
| --- | --- | --- | --- |
| **HTTP** | 웹 통신(브라우저) | TCP | **80** |
| **HTTPS** | 보안 웹 통신(SSL/TLS 적용) | TCP | **443** |
| **FTP** | 파일 전송 | TCP | **21 (제어)** / **20 (데이터)** |
| **SSH** | 원격 접속(암호화) | TCP | **22** |
| **TELNET** | 원격 접속(암호화 없음) | TCP | **23** |
| **SMTP** | 메일 전송 | TCP | **25** |
| **POP3** | 메일 수신(다운로드 방식) | TCP | **110** |
| **IMAP** | 메일 수신(서버 저장 방식) | TCP | **143** |
| **DNS** | 도메인 → IP 변환 | UDP(기본), TCP(특수) | **53** |
| **DHCP** | IP 자동 할당 | UDP | **67(서버)** / **68(클라이언트)** |
| **NTP** | 시간 동기화 | UDP | **123** |
| **LDAP** | 디렉터리 서비스 | TCP/UDP | **389** |
| **RDP** | 원격 데스크탑 | TCP | **3389** |
| **MySQL** | 데이터베이스 | TCP | **3306** |
| **PostgreSQL** | 데이터베이스 | TCP | **5432** |

> 쓰다보면 포트번호는 저절로 외워지니 너무 암기하는데 열 올리지 말 것 ❗

## 4️⃣ OSI 5·6·7 요약

| 계층 | 역할 | PDF 기준 핵심 내용 |
| --- | --- | --- |
| **L5 – 세션(Session)** | 응용 간 통신 세션 생성·유지·종료 | TCP/IP 세션 개념 관리 |
| **L6 – 표현(Presentation)** | 데이터 포맷, 인코딩, 암호화/복호화 | GIF, JPEG, ASCII 등 포맷 정의 |
| **L7 – 응용(Application)** | 애플리케이션·사용자에게 네트워크 기능 제공 | HTTP, FTP, SSH 등 |

## 5️⃣ 전체 흐름 관점에서 보는 OSI 상위 계층

데이터가 실제 사용자에게 도달하기까지 상위 계층은 다음과 같이 작동합니다.

  1. **응용 계층(L7)**
→ 사용자가 HTTP 요청을 보냄

  1. **표현 계층(L6)**
→ 데이터 포맷 결정(예: UTF-8, JPEG)

  1. **세션 계층(L5)**
→ 서버와 논리적 연결(세션) 유지

  1. 이후 L4~L1을 통해 전송 후 다시 반대로 올라감
즉,

> "사용자 요청 → 데이터를 보기 좋게 포맷 → 연결을 만들고 유지 → 전송"

의 흐름을 담당하는 것이 Layer 5·6·7입니다.

---

## 6️⃣ 마무리

5·6·7 계층은 사용자가 실제로 사용하는 서비스들이 동작하는 영역이며, 각 프로그램은 **정해진 포트 번호를 통해 네트워크에서 구분되어 사용된다**는 정도로 이해하면 충분할 거 같습니다.
