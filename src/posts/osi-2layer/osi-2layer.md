## 0️⃣ Layer 1에서 Layer 2로

앞서 물리 계층에서 정리한 것처럼, 네트워크의 가장 아래단에서는 데이터가 단순한 **전기 신호 형태(0과 1)** 로만 존재했습니다.

하지만 이 신호를 받아들이는 장치 입장에서는 그게 어떤 의미를 가지는지 알 수 없죠.

그래서 등장하는 것이 바로 **데이터링크 계층(Data Link Layer)** 입니다.

이 계층은 물리적으로 흘러온 신호를 **의미 있는 데이터 단위인 ‘프레임(Frame)’으로 구성하고**, 오류 없이 정확히 전달될 수 있도록 관리합니다.

---

## 1️⃣ OSI 7계층 — L2 : 데이터링크 계층이란?

![image.png](/images/posts/osi-2layer/image1.png)

데이터링크 계층은 OSI 7계층 중 두 번째에 위치하며, **‘안정적인 정보 전달’을 담당하는 계층**입니다. 즉, 물리 계층에서 올라온 비트 신호를 ‘데이터’로서 신뢰할 수 있는 형태로 정리하고 제어하는 단계입니다.

핵심 역할은 다음과 같습니다.

  - **오류 제어(Error Control)** : 전송 중 오류를 검출하고 재전송을 요청
  - **흐름 제어(Flow Control)** : 송수신 속도를 조절하여 데이터 유실 방지
  - **프레임 동기화(Frame Synchronization)** : 송신자와 수신자가 프레임의 경계를 인식
  - **주소 지정(Addressing)** : 송수신 장치를 MAC 주소로 식별
  - **매체 접근 제어(MAC Control)** : 여러 장치가 동시에 전송하려 할 때 충돌을 방지
---

## 2️⃣ 데이터 전송 단위 — 프레임(Frame)

데이터링크 계층의 전송 단위는 **프레임(Frame)** 입니다.

프레임은 상위 계층(네트워크 계층)에서 받은 **패킷(Packet)** 을 받아 MAC 주소, 오류 검출 정보 등을 추가한 구조로, 물리 계층으로 내려가기 전에 완성되는 데이터의 “전송 단위”입니다.

```Plain Text
[Header] + [Payload] + [Trailer]


```

| 구분 | 구성 요소 | 설명 |
| --- | --- | --- |
| **Header** | 수신지 MAC 주소 / 송신지 MAC 주소 / 타입 정보 | 프레임의 목적지와 출발지를 구분 |
| **Payload** | 실제 데이터(상위 계층에서 받은 패킷) | 전송할 데이터 내용 |
| **Trailer** | FCS(Frame Check Sequence) | CRC 기반 오류 검출용 값 |

> 💡 

> **FCS(Frame Check Sequence) ❓ **넌 누구냐,,,
> FCS는 프레임이 전송 중 손상되었는지 확인하기 위한 필드입니다.
> CRC(Cyclic Redundancy Check, 순환 중복 검사) 방식을 사용하여 오류를 탐지합니다.

> 💡 

> **CRC(Cyclic Redundancy Check, 순환 중복 검사)** ❓ 넌 또 뭐야,,,

이건 프레임 안의 데이터에 **특정 규칙(다항식)을 적용해 계산한 검증 값**이에요. 수신 측에서도 동일한 계산을 수행했을 때 결과가 다르면, “전송 중 데이터가 손상되었구나” 하고 판단하게 됩니다.

쉽게 말해, **데이터 전송 중 비트 하나라도 바뀌었는지 잡아내는 수학적 검사 방식**입니다.

---

## 3️⃣ MAC 주소와 장비

데이터링크 계층에서 가장 중요한 개념 중 하나가 **MAC 주소**입니다.

MAC(Media Access Control) 주소는 네트워크 장비(NIC)에 부여된 **고유한 하드웨어 주소**로, LAN 내에서 통신할 때 송신지와 수신지를 식별하는 데 사용됩니다.

  - **형식:** 6바이트(48비트)
  - **표현:** 16진수 12자리 (예: `00:1A:2B:3C:4D:5E`)
MAC 주소는 **같은 네트워크 대역 내에서만 유효**하며, 다른 네트워크로 넘어가면 IP 주소가 사용됩니다.

이 역할 구분이 바로 **2계층(데이터링크) ↔ 3계층(네트워크)** 의 경계라고 할 수 있습니다.

---

## 4️⃣ 스위치(Switch)와 브리지(Bridge)

데이터링크 계층에서 실제로 동작하는 대표 장비는 **스위치(Switch)** 와 **브리지(Bridge)** 입니다.

### 🔹 스위치(Switch)

스위치는 허브와 달리 **정확한 목적지(MAC 주소)** 를 파악해 필요한 포트로만 데이터를 전송합니다. 이를 통해 네트워크 충돌을 줄이고 효율을 높입니다.

### 🔹 브리지(Bridge)

브리지는 두 개의 네트워크 세그먼트를 연결하는 장치로, 프레임의 목적지 MAC 주소를 보고 전달 방향을 결정합니다. 즉, 동일 네트워크 안에서는 필터링하고, 다른 네트워크로 보낼 필요가 있을 때만 중계합니다.

---

## 5️⃣ 오류 제어와 흐름 제어

데이터링크 계층은 단순히 데이터를 전달하는 것이 아니라,

전송 중 문제가 생기면 **스스로 점검하고 조정**하는 역할도 수행합니다.

  - **오류 제어(Error Control)** : FCS를 이용해 손상된 프레임을 검출하고 재전송 요청
  - **흐름 제어(Flow Control)** : 수신 장치의 처리 속도에 맞게 전송 속도를 조절
  - **순서 제어(Sequence Control)** : 프레임 순서를 유지해 데이터 일관성 확보
이러한 기능 덕분에,

상위 계층(예: TCP/IP의 네트워크 계층)은 **전송의 신뢰성 문제를 신경 쓰지 않아도 됩니다.**

데이터링크 계층이 **같은 네트워크 구간 내(Local link) **에서 프레임 단위의 오류 제어와 흐름 제어를 수행하기 때문에, 상위 계층은 **논리적인 데이터 전달에만 집중할 수 있습니다.**

---

## 6️⃣ 데이터링크 계층의 하위 구성 — LLC와 MAC

데이터링크 계층은 내부적으로 **두 개의 하위 계층**으로 나뉩니다.

| 하위 계층 | 설명 |
| --- | --- |
| **MAC (Media Access Control)** | 네트워크 매체 접근 제어, 충돌 방지, 주소 지정 |
| **LLC (Logical Link Control)** | 오류 검출, 흐름 제어, 상위 계층과의 인터페이스 |

  - **MAC 계층**은 실제 네트워크 매체에 접근해 데이터를 송수신을 담당합니다.
  - **LLC 계층**은 상위 계층(네트워크 계층)과의 논리적 연결을 담당합니다.
즉, MAC이 “물리적 전송”을 다루고 LLC가 “논리적 연결”을 담당한다고 이해하면 쉽습니다.

---

## 7️⃣ 마무리 정리

데이터링크 계층은 **물리적으로 전달된 신호를 ‘의미 있는 데이터’로 바꾸는 첫 번째 단계**입니다.

프레임 단위로 데이터를 관리하고, 오류를 검출하며, 흐름을 제어해 안정적인 통신을 보장합니다.

어렵게 생각하지 말고, “같은 네트워크 안에서 데이터를 안전하게 주고받는 계층”이라고 이해하면 됩니다.
